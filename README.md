# DIO-Ciberseguran-a-
Simulando um Ataque de Brute Force de Senhas com Medusa e Kali Linux

Nesse desafio, a ideia foi simular ataques de força bruta em um ambiente controlado. Usei o Kali Linux junto com a ferramenta Medusa pra testar vulnerabilidades em máquinas como Metasploitable 2 e DVWA.

O que eu aprendi

Basicamente: **senha fraca é um convite pra invasão**.

Durante os testes, deu pra ver como ferramentas automatizadas conseguem testar várias combinações de login e senha MUITO rápido. Usando wordlists simples, já foi possível encontrar acessos vulneráveis sem muito esforço.

Ou seja, não precisa ser nenhum “hacker nível filme” — se o sistema tiver senha fraca, ele já tá em risco.

Também entendi que essas ferramentas não são só pra ataque. Na real, elas são muito usadas pra **testar a segurança** e encontrar falhas antes que alguém mal-intencionado explore.

 Como evitar esse tipo de problema

Depois dos testes, ficou bem claro o que precisa ser feito:

* Criar senhas fortes (nada de "123456" )
* Não repetir senha em vários lugares
* Bloquear login depois de várias tentativas erradas
* Usar autenticação em dois fatores (MFA)

Conclusão

Esse desafio mostrou que o básico bem feito já faz muita diferença. Segurança não começa com coisas super complexas, mas sim com boas práticas simples.

Se a base for fraca, qualquer ferramenta simples já consegue explorar isso.

 Ferramentas usadas

* Kali Linux
* Medusa
* Metasploitable 2
* DVWA.
